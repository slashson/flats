# Studio QR site

Минималистичный mobile-first сайт-«гид» по студии short-stay. Открывается по QR-коду в квартире.

Без бэкенда, без сборки, без зависимостей — статика на чистом HTML/CSS/JS.

---

## 📂 Структура

```
.
├── index.html        # все экраны в одном файле
├── style.css         # стили
├── app.js            # навигация, i18n, копирование, карты
├── config.json       # ⬅️ данные студии (Wi-Fi, контакты, цены...)
├── i18n/
│   ├── ru.json       # ⬅️ тексты на русском
│   ├── uk.json       # ⬅️ тексты на украинском
│   └── en.json       # ⬅️ тексты на английском
└── images/           # фото техники (опционально)
```

---

## 🚀 Локальный запуск

`fetch()` не работает с `file://`, поэтому нужен простой HTTP-сервер. Любой из этих способов:

```bash
# Python (есть на macOS из коробки)
python3 -m http.server 8000

# Node.js
npx serve .
```

Открыть [http://localhost:8000](http://localhost:8000).

## ✏️ Как менять контент

### Данные студии — `config.json`

```jsonc
{
  "studioNumber": "12",            // номер на главном экране
  "wifi": {
    "ssid": "MyStudio_5GHz",       // Wi-Fi сеть
    "password": "welcome2024"      // Wi-Fi пароль
  },
  "contacts": {
    "telegram": "admin_placeholder", // юзернейм без @
    "phone": "+380000000000"         // в международном формате
  },
  "location": {
    "lat": 50.4501,                // координаты студии
    "lng": 30.5234,                //  → для «Полезное рядом»
    "address": "Киев"
  },
  "prices": {
    "extensionPerNight": 500,
    "deposit": 2000,
    "currency": "грн"
  },
  "checkoutTime": "12:00"
}
```

### Тексты — `i18n/ru.json`, `i18n/uk.json`, `i18n/en.json`

Все тексты сайта на каждом языке. Структура одинаковая, ключи совпадают — если меняете один файл, **поменяйте и в остальных двух**.

Пример (правила проживания, добавить новый пункт):

```jsonc
"rules": {
  "list": [
    "Курение в студии запрещено",
    "Новое правило...",       // ⬅️ просто добавили строку
    ...
  ]
}
```

### Редактирование через GitHub (без терминала)

1. Открыть файл (например, `config.json`) на github.com.
2. Нажать карандашик ✏️ справа сверху.
3. Поменять текст, нажать **Commit changes**.
4. Через ~30 секунд сайт обновится автоматически.

Сейчас в разделе «Как пользоваться» стоят серые заглушки.

1. Добавьте файлы в `images/` (например, `boiler.jpg`).

## 🌍 Деплой на GitHub Pages

1. Создать новый репозиторий на github.com.
2. Залить файлы (через web UI или `git push`).
3. **Settings → Pages → Source:** `Deploy from a branch`, branch `main`, folder `/ (root)`.
4. Через минуту сайт доступен по адресу `https://<username>.github.io/<repo>/`.

Можно подключить свой домен (Settings → Pages → Custom domain).

## 📱 QR-код

Когда домен сайта известен (например, `https://studio.example.com`):

- Любой бесплатный генератор, например [qrcode-monkey.com](https://www.qrcode-monkey.com/) или [qr-code-generator.com](https://www.qr-code-generator.com/).
- В URL зашиваем чистый домен (без `?utm_…`, без хеша) — самое лаконичное.
- Распечатать на матовой бумаге размером ~5×5 см, разместить на видном месте в студии (на двери, у входа, на столе).

---

## ➕ Несколько студий в будущем

Когда появится вторая студия, переключаемся на multi-studio за 30 минут:

1. `config.json` → `studios.json` со списком студий по ID.
2. URL: `?s=5` → JS читает параметр и подставляет нужную студию.
3. QR на каждую студию ведёт на свой URL.

Архитектура уже готова к этому — все данные читаются из одного объекта.

## 🧪 Что протестировать

- iPhone Safari (главная цель)
- Android Chrome
- Кнопка «Скопировать пароль» — должен показаться тост
- Кнопки Telegram — должны открывать приложение Telegram
- Кнопка «Позвонить» — должна открывать звонилку
- Переключатель языка сохраняется после перезагрузки
- Раздел «Полезное рядом» — карта открывается с поиском вокруг адреса студии
